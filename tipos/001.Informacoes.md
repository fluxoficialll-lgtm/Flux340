📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A pasta `tipos` centraliza todas as definições de tipos e interfaces TypeScript do projeto.
O objetivo é garantir a consistência, a segurança e a previsibilidade dos dados em toda a aplicação.
Cada arquivo deve agrupar tipos relacionados a um contexto ou funcionalidade específica (ex: `types.Autenticacao.Login.ts`).
Use interfaces para definir a forma de objetos e `type` para uniões, tuplas ou tipos mais complexos.

🔄 Fluxo de Dados 🔄
─────────────────────────────
Os tipos definidos nesta pasta são importados por componentes, hooks, serviços e outras partes do código.
Não há um "fluxo de dados" no sentido de execução. A pasta serve como uma fonte de verdade para a estrutura dos dados.
O compilador TypeScript utiliza esses tipos em tempo de desenvolvimento para verificar a corretude do código.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de Tipagem de Dados: Definir a estrutura exata de objetos, parâmetros de funções e retornos.
Responsabilidade de Contrato de Dados: Servir como um contrato entre diferentes partes da aplicação (ex: frontend e backend, ou entre dois serviços).
Responsabilidade de Segurança de Tipo: Ajudar a prevenir erros comuns em tempo de execução, como `undefined is not a function` ou acesso a propriedades inexistentes.
Responsabilidade de Documentação: Os próprios tipos servem como uma forma de documentação, descrevendo quais dados são esperados.
