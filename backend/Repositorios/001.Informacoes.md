# Documentação da Camada de Repositórios (Backend)

O diretório `/backend/Repositorios` representa a camada de acesso a dados mais fundamental e de mais baixo nível da aplicação. Sua única responsabilidade é interagir diretamente com o banco de dados, executando as queries SQL puras.

## Estrutura e Organização

A nomenclatura dos arquivos segue o padrão já estabelecido, espelhando os domínios das camadas superiores:

`Repositorio.{Dominio}.{Subdominio}.js`

### Principais Responsabilidades:

- **Execução de Queries:** Cada função dentro de um arquivo de repositório (`criar`, `obterPorId`, `atualizar`, `remover`, etc.) corresponde a uma operação atômica no banco de dados. Elas recebem os dados já prontos e os utilizam para construir e executar uma query SQL (INSERT, SELECT, UPDATE, DELETE).

- **Abstração do Banco de Dados:** Esta camada abstrai os detalhes de implementação do banco de dados do resto da aplicação. Se no futuro o banco de dados fosse trocado (de PostgreSQL para MySQL, por exemplo), esta seria a única camada que precisaria de modificações diretas nas queries.

- **Nenhuma Lógica de Negócio:** É crucial que os repositórios não contenham **nenhuma** lógica de negócio, validação de permissões ou orquestração. Eles simplesmente executam a tarefa que lhes foi solicitada pela camada superior (geralmente a camada de `Consultas` ou `Serviços`).

## Fluxo de Dados

1.  A camada de `Serviços` ou `Consultas` determina que uma operação de banco de dados é necessária.
2.  Ela chama a função apropriada no `Repositorio` correspondente, passando os dados necessários (ex: os valores para uma nova linha a ser inserida).
3.  A função do repositório constrói a string da query SQL.
4.  Ela utiliza o driver do banco de dados (neste caso, o `pool` do `pg`) para executar a query.
5.  O resultado bruto do banco de dados (as linhas ou o status da operação) é retornado para a camada que a chamou.
